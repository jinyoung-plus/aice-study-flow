# AICE Associate 학습 도구

> 모두를 위한 AICE Associate 시험 대비 학습 웹앱

🔗 **라이브 데모**: https://aice-study-flow.lovable.app

## 한 줄 요약

AICE Associate(KT·한국경제신문 공동주관) 국가공인 자격시험을 워크플로우 기반으로 학습할 수 있는 도구. 별칭 + 4자리 비밀번호로 가볍게 가입하고, "데이터분석 → 전처리 → 모델링" 3단계를 따라 체계적으로 학습한다.

## 왜 만들었는가

### 1. 국가공인 인공지능 활용능력 시험 학습을 돕기 위해

AICE Associate는 KT와 한국경제신문이 공동 주관하는 국가공인 인공지능 활용능력 시험이다. 시험을 준비하는 누구나 부담 없이 활용할 수 있는 공개 학습 도구를 만드는 것이 목표다.

### 2. 시험의 본질을 따라가는 워크플로우 설계

AICE Associate는 일반적인 ML 강의 커리큘럼과 달리 **실제 분석 워크플로우**(데이터 로딩 → 전처리 → 모델링)를 따라 코드를 짜는 실습형 시험이다. 학습 도구도 이 본질을 그대로 따르게 설계했다. 카테고리식 카드가 아니라, 3단계 파이프라인을 시각적으로 드러내는 구조다.

### 3. 누구나 부담없이 시작할 수 있는 라이트 식별 설계

공개 사이트이므로 **개인정보를 수집하지 않는 라이트 식별 방식**을 선택했다. 별칭(가명 권장) + 4자리 비밀번호 조합은 보안 강도가 약하지만, 저장되는 데이터가 학습 진도뿐이라는 점에서 합리적인 트레이드오프라고 판단했다. 보안 한계는 사용자에게 명확히 고지한다(별칭은 가명 권장, 비밀번호는 다른 곳과 다른 4자리 사용 권장).

## 무엇을 만들었는가

### 6개 화면

1. **랜딩** — 사이트 소개 + 시작하기/로그인 진입
2. **가입·로그인** — 별칭 + 4자리 비밀번호 (라이트 식별)
3. **학습 대시보드** — 3단계 워크플로우 카드 + 누적 통계 (총 풀이 문제·평균 정답률·학습 일수)
4. **영역별 학습** — 치트시트 / 플래시카드 / 빈칸채우기 / 퀴즈 4탭
5. **마이페이지** — 영역별 정답률·약점 영역 추천
6. **관리자 화면** — 사용자 목록 + 학습 이력 초기화·계정 삭제

### 7개 테이블 DB 스키마

- `users` · `stages` · `subtopics` · `questions` · `flashcards` · `cheatsheets` · `user_progress`
- 비밀번호는 bcrypt로 해시 후 저장
- 학습 이력은 시도마다 한 행씩 누적 (시간순 학습곡선 시각화 가능)

### 학습 콘텐츠 구조

- **3단계**: 데이터분석 · 전처리 · 모델링
- **14개 sub-topic** 전체에 콘텐츠 시드 완료
  - 데이터분석(5): 라이브러리 · 파일로드 · 데이터 조작(groupby/merge/concat) · 단변수 시각화 · 이변수 시각화
  - 전처리(5): 이상치(IQR) · 결측치 · 원핫 인코딩 · 데이터 분리(train_test_split) · 스케일링
  - 모델링(4): ML학습(DecisionTree/RandomForest) · 변수중요도 시각화 · 성능평가 · 딥러닝(Sequential) · 학습곡선
- 각 sub-topic마다 **4탭 학습**: 치트시트 / 플래시카드 / 빈칸채우기 / 퀴즈
- 시험장 함정 포인트(예: `df.get_dummies()` 금지, `fillna(inplace=True)` 금지, `stratify=y`는 분류 전용, `feature_importances_` 끝 언더스코어 등)를 퀴즈·해설에 반영

## 데모 사용법

**사이트 주소**: https://aice-study-flow.lovable.app

### 일반 사용자

- 별칭(아무거나) + 4자리 비밀번호로 가입
- 학습 대시보드에서 단계 카드 클릭 → 영역별 4탭 학습 진행
- 풀이 결과가 자동으로 진도·정답률에 반영

### 관리자

- 별칭: `admin`
- 비밀번호: `!@#$`
- 관리자 화면에서 전체 사용자 조회 + 학습 이력 초기화 / 계정 삭제 가능

> ⚠️ 데모 시연용 단순 자격증명. 실제 운영 시 환경변수화 필요.

## 기술 스택

- **프론트엔드**: React + TypeScript, Tailwind CSS, React Router, recharts
- **백엔드**: Supabase (PostgreSQL · 인증 · Edge Functions)
- **배포**: Lovable Cloud

## 한계 및 향후 계획

### 현재 한계

- 라이트 식별 방식이라 비밀번호 복구 기능 없음 (분실 시 새 별칭으로 재가입)
- 관리자 자격증명이 코드에 노출됨 (데모 한정)
- 모바일 환경에서 일부 코드 블록의 가독성이 제한적

### 향후 계획

- 환경변수 기반 관리자 자격증명으로 전환
- 학습곡선 시각화(시간순 정답률 그래프) 추가
- 사용자 간 학습 진도 비교 기능
- sub-topic별 콘텐츠 추가 보강 (각 영역 문제 수 확대)
